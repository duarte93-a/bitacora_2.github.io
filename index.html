El modulo flex-box

El módulo flex-box (contenedor flexible / flexible box) tiene como objetivo proporcionar una manera más eficiente de diseñar, alinear y distribuir el espacio entre los elementos en un contenedor, incluso cuando su tamaño es desconocido o dinámico, usando por lo tanto la palabra flex. La idea principal consta de otorgar al diseño “flexibilidad” es decir la capacidad de poder modificar su ancho, alto y el orden de los elementos para llenar mejor los espacios designados (esto es útil cuando desconocemos el tamaño y proporciones del dispositivo al cual el usuario tiene, permitiéndole su adaptación a cualquier tipo de pantalla). Un contenedor flexible expande los elementos para llenar el espacio libre disponible o los reduce para evitar el desbordamiento. Lo mas importante es que el diseño de flex-box es independiente de la dirección en lugar de los diseños regulares, si bien estos funcionan bien para las páginas, carecen de flexibilidad en aplicaciones de gran tamaño y complejidad, en especial cuando se trata de cambio de orientación, tamaño, estiramiento, encogimiento etc.

Soportes de navegador

linea de tiempo

  • Display: flex; (sintaxis recién de la especificación).
  • Display: flexbox; (sintaxis no oficial de 2011).
  • Display: box; (sintaxis antigua de 2009).

Nota: este modulo actualmente se encuentra como candidato recomendable del w3c a partir de octubre de 2017.

Propiedades del padre de un contenedor flexible

Display:

Esto define un contenedor flexible; en línea o bloque en función del valor dado. Permite un contexto flexible para todos sus hijos directos.

.container {display: flex; (0 inline-flex)}

Flex-direction:

Esto establece el eje principal, definiendo la dirección en que se colocan los elementos flexibles en el contenedor, flex-blox es un conjunto de diseño de una sola dirección. Estos pueden ser en elementos horizontales o verticales.

.container {flex-direction: row / row-reverse/ column/ column-reverse;}

Seleccion de display

Flex-wrap

Por defecto, todos los elementos intentaras encajar en una línea. Puede cambiar eso y permitir que los elementos se ajusten según sea necesario con esta propiedad.

.container {flex-wrap: nowrap /wrap/wrap-reverse;}

Selección de flex wrap

flex-flow

Esta es una abreviatura de propiedad de flex-direction y flex-wrap, que juntas definen los ejes principales y transversal del contenedor flexible, el valor que esta determinado es la de row noweap.

Flex-flow: <”flex-direction”> // <”flex-wrap> /p>

justify-content

Esto define la alineación a lo largo del eje principal, ayudando a distribuir el espacio libre adicional que sobra cuando todos los elementos flexibles de una línea son inflexibles o flexibles, pero han alcanzado su tamaño máximo, además de ejercer cierto control sobre la alineación de los elementos cuando se desbordan en la línea.

Selección de justify-content

Align-items

Esto define el comportamiento predeterminado de como los elementos flexibles se distribuyen a lo largo del eje trasversal en la línea actual. pensé en ello como la versión de justify-content para el eje trasversal.


.conteiner { align-items: stretch/ flex-start / flex-end / center/ baseline}

Selección de Align-items

Aling-content

Esta propiedad alinea los renglones de un contenedor flexible cuando hay espacio extra en el eje trasversal, de manera similar a como justify-content alinea elementos individuales dentro del eje principal.


.container { aling-content: flex-start / flex-end/ center/ space-between / space-araund / stretch;}

Nota esta propiedad no tiene efecto cuando solo hay una línea de elementos flexibles.

Selección de Align-content

grap

la propiedad gap controla el espacio entre elementos flexibles. Se aplica ese espacio solo entre elementos que no están en los bordes exteriores.

Gap:10px 20px

Las propiedades para los hijos de flex-blox

orden

Los elementos flexibles se colocarán según el orden donde los colocamos, sin embargo, la propiedad orden controla el orden en el que aparece en el contendedor flexible


. ítem {orden: número entero;/*por default es 0*/}

Nota: Todos los elementos del contenedor tienen 0 por lo que podemos acomodarlos en el orden que deseamos, podemos usar además números negativos.

Flex-glow

Define la capacidad para un ítem crezca si es necesario, este acepta un valor entero que sirve como proporción, indicando la cantidad de espacio disponible dentro del contenedor flexible debe ocupar el ítem. El numero entero permite el crecimiento en unidades. Este siempre comienza en cero.

. item {Flex-grow: numero entero; /*por default es 0*/}

Nota Los números negativos no son válidos, flex-grow 0 no permite el crecimiento más allá de los pixeles designados.

flex-shink

Especifica la capacidad de un elemento para reducir en relación con el resto de los elementos flexibles del contenedor

.item{ flex shirink; numero entero/* por default es 1*/}

flex basis

Esto define el tamaño predeterminado de un elemento antes de distribuir el espacio restante. Puede ser una longitud (por ejemplo 20%,5rem, etc.). o una palabra clave

Flex-basis: numero / auto/innitial/inherit;

Las palabras clave puede ser:

Flex-bases:fill/max-content/main.content/fit-content;

Nota si se establece en 0, el espacio adicional alrededor del contenido no se toma en cuenta. Si se configura en auto, el espacio adicional se distribuye según su valor de flex-grow

Aling-self

Esto permite que la alineación predeterminada (o la especificada por align-items) se anule para los elementos flex específicos, tenga en cuenta que las propiedades float, clear y vertical-aling no tienen efecto en un elemento flexible.

.flex-item{ align-self; flex-start / flex-end / center / baseline / stretch}
